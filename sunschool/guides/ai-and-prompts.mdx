## Prompt transparency

Sunschool's core principle: **the parent owns the prompt**. Every instruction sent to the AI is visible in the source code. Nothing is hidden behind proprietary prompts.

All prompt templates live in `server/prompts/grades/` and are organized by grade band:

| Grade band | File | Target audience |
|------------|------|-----------------|
| K-2 | `gradeK2.ts` | Kindergarten through 2nd grade |
| 3-4 | `grade3_4.ts` | 3rd and 4th grade |
| 5-6 | `grade5_6.ts` | 5th and 6th grade |
| 7-8 | `grade7_8.ts` | 7th and 8th grade |
| 9+ | `grade9plus.ts` | High school and above |

Each prompt template controls:

- **Vocabulary complexity** — Simpler words for younger students
- **Sentence length** — Shorter sentences for early readers
- **Content depth** — Age-appropriate explanations
- **Question difficulty** — Quiz questions matched to reading level

## Content validation

After the AI generates a lesson, it passes through validation:

- **Placeholder detection** — Rejects generic content like "This is a lesson about Math"
- **Structure checks** — Must have a title, 2+ sections, and 2+ quiz questions
- **Question validation** — Each question needs 2+ options and a valid correct answer
- **Retry on failure** — Up to 3 generation attempts before returning an error

The validator lives in `server/services/lesson-validator.ts`. If all attempts fail, the server returns a 503 error — it never saves stub content to the database.

## AI providers

| Provider | Purpose | Config |
|----------|---------|--------|
| **OpenRouter** (primary) | Lesson text, quiz questions, SVG illustrations | `OPENROUTER_API_KEY` |
| **Perplexity** (optional) | Knowledge context enrichment | `PERPLEXITY_API_KEY` |
| **Bittensor** (experimental) | Decentralized LLM alternative | `BITTENSOR_API_KEY`, `LLM_PROVIDER=bittensor` |

Provider selection is handled by `server/services/ai.ts` with automatic fallback from Bittensor to OpenRouter.

## Adaptive learning

Sunschool tracks concept mastery to personalize future lessons:

- **Concept tagging** — Each quiz question is tagged with the concepts it tests
- **Mastery tracking** — Per-concept accuracy tracked across all quiz attempts
- **Question deduplication** — SHA-256 hashing prevents repeat questions within a 30-day window
- **Spaced repetition** — Weak concepts are prioritized in future quiz generation
- **Subject recommendations** — Category mapping suggests next topics based on performance

## Auditing the prompts

To see exactly what the AI is told:

1. Clone the repo: `git clone https://github.com/allonethingxyz/sunschool.git`
2. Browse `server/prompts/` for all prompt templates
3. Browse `server/services/enhanced-lesson-service.ts` for the generation pipeline
4. Browse `server/services/svg-llm-service.ts` for illustration generation prompts
