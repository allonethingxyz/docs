## Getting started

1. Fork [allonethingxyz/sunschool](https://github.com/allonethingxyz/sunschool)
2. Clone your fork locally
3. Install dependencies: `npm install`
4. Create a `.env` file (see [self-hosting](/sunschool/self-hosting))
5. Run in development mode: `npm run dev`

## Pull requests

All PRs require review before merging to `main`. The `main` branch is protected.

1. Create a feature branch from `main`
2. Make your changes
3. Run type checking: `npx tsc --noEmit`
4. Run tests: `npm test`
5. Submit a pull request

## Repository structure

There are two repositories:

| Repo | Purpose |
|------|---------|
| [allonethingxyz/sunschool](https://github.com/allonethingxyz/sunschool) | Public, open source, branch-protected `main` |
| Private deployment repo | Railway auto-deploys from this, not public |

Contributors work with the public repo. The deployment repo tracks upstream and adds deployment-specific configuration.

## Code style

- TypeScript throughout (client and server)
- React functional components with hooks
- Drizzle ORM for database queries
- Express.js middleware pattern for auth and role checks
- `asyncHandler` wrapper for async route handlers

## Key conventions

- **Single spec column** — Lessons use one `EnhancedLessonSpec` stored as JSON
- **No stubs** — Generation failures return 503, never save placeholder content
- **Background tasks** — Image generation runs async via `backgroundTask()` helper
- **Model fallbacks** — AI services try multiple models before failing

## Reporting issues

File issues at [github.com/allonethingxyz/sunschool/issues](https://github.com/allonethingxyz/sunschool/issues).
