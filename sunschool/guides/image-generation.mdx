## Overview

Every lesson includes AI-generated educational illustrations. Images are generated in the background after the lesson text is ready, so learners can start reading immediately.

## Image providers

The `IMAGE_PROVIDER` environment variable controls the primary strategy:

| Value | Behavior |
|-------|----------|
| `svg-llm` (default) | Generate SVG illustrations via LLM |
| `openrouter` | Generate raster images via multimodal models |
| `stability` | Use Stability AI API |

## SVG LLM service

The default provider (`svg-llm`) generates SVGs directly from a language model. The LLM is prompted to output valid SVG markup with strict rules:

- Only inline styles or presentation attributes (no `<style>` or `<script>` tags)
- No external references
- No event handlers
- Child-safe, educational content only
- Clean white or light backgrounds
- Readable fonts (Arial, sans-serif)

### Model fallback chain

The service tries models in order until one succeeds:

1. **Primary model** — `OPENROUTER_SVG_MODEL` (default: `google/gemini-3.1-pro-preview`)
2. **Configured fallbacks** — `SVG_MODEL_FALLBACKS` env var (comma-separated)
3. **Built-in defaults** — `google/gemini-3.1-flash-lite-preview`, `google/gemini-3-flash-preview`

Models that return **402** (insufficient credits) or **404** (not found) are automatically skipped.

## SVG sanitization

All generated SVGs pass through a server-side sanitizer before storage:

1. **Forbidden tags removed** — `script`, `style`, `iframe`, `object`, `embed`, `form`, `input`, `link`, `meta`
2. **Event handlers stripped** — All `on*` attributes
3. **Dangerous URIs removed** — `javascript:` and `data:` in `href`/`src` attributes
4. **Content validation** — SVGs must contain at least 2 meaningful drawing elements (not just a background rectangle)

## Diagram types

The diagram generator supports several types:

| Type | Description |
|------|-------------|
| `flowchart` | Rectangular boxes connected by arrows |
| `concept map` | Oval nodes with labeled relationship lines |
| `cycle` | Circular stages connected by curved arrows |
| `comparison` | Side-by-side or Venn diagram layout |
| `hierarchy` | Tree diagram with levels |
| `timeline` | Horizontal timeline with labeled events |

## Client-side rendering

The frontend renders images with a fallback chain:

1. **base64Data** — Inline raster image
2. **svgData** — Inline SVG markup (rendered via `dangerouslySetInnerHTML`)
3. **path** — Server-hosted image file
4. **Placeholder** — Animated gradient placeholder while images load

The client polls every 5 seconds until real images are available, using React Query's `refetchInterval`.

## Configuration

```env
# Primary image strategy
IMAGE_PROVIDER=svg-llm

# SVG model selection
OPENROUTER_SVG_MODEL=google/gemini-3.1-pro-preview
SVG_MODEL_FALLBACKS=google/gemini-3.1-flash-lite-preview,google/gemini-3-flash-preview

# Limits
IMAGE_GENERATION_TIMEOUT=15000
MAX_IMAGES_PER_LESSON=4
```
